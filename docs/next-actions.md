# 直近1週間の着手タスク

ゴール逆算の起点。最初の1週間で「基盤」を作る。

## Day 1-2：プロフィール＆動線設計

- [ ] Threadsプロフィール文を作成（Instagram代行/コンサル業者向けと明示）
- [ ] LP（or 簡易ペライチ）作成、URLをThreadsプロフィールに設定
- [ ] DM返答テンプレ用意（問い合わせ受領→ヒアリング→商談予約の流れ）
- [ ] **新規Instagram垢の開設**（既存個人垢ではなくBtoB専用）
  - アカウント名：「○○｜Instagram運用代行の集客支援」など職能明示
  - プロフィール：ターゲット明記
  - ハイライト：実績／よくある質問／お客様の声／無料相談導線
  - 投稿コンテンツ10件先行ストック（代行業者の悩み解決系）
  - リンク：LP / 無料相談DM / Lit.linkで導線まとめ
- [ ] 新規Instagram垢ベースで新規Threads垢を開設

## Day 3-4：ナレッジ再分類

- [ ] 既存ナレッジを5軸に振り分け
  - `knowledge/acquisition/` - 顧客獲得
  - `knowledge/efficiency/` - 効率化・AI活用
  - `knowledge/pricing/` - 単価UP
  - `knowledge/trends/` - 業界トレンド
  - `knowledge/confessions/` - 失敗談・本音
- [ ] 各軸20ネタ以上ストックを確認（足りない軸は補強）
- [ ] `knowledge/voice.md` に自分の口調サンプル10件分メモ
- [ ] `knowledge/used_log.json` を初期化（空配列）

## Day 5-6：パイプライン最小構成構築

- [ ] スクリプト：ナレッジ → Claude API → コンソール出力（投稿前の確認用）
- [ ] Threads API のアクセストークン発行
- [ ] SQLite スキーマ設計（posts, daily_stats テーブル）
- [ ] 3日間は「生成→人間が確認してから手動投稿」で品質チェック

## Day 7：本格稼働判断

- [ ] Threads API投稿の自動実行スタート（1日1投稿から）
- [ ] インサイト記録の自動化（毎晩23:00 cron）
- [ ] 翌週分のスケジュール確認

## 注意事項

### 新規アカウント開設時のシャドウバン対策

- 最初の2週間は活動量を抑える（1日1投稿、過剰なフォローやいいねNG）
- 段階的に投稿頻度を上げる
- インスタ側は最初は週2〜3投稿で十分（Threadsがメイン集客口）

### 品質ゲート

- 自動投稿開始前に最低10件は手動レビュー
- AI臭がないか、ターゲットに刺さるか、自分の口調と合っているかを確認
- ズレを見つけたらプロンプト or voice.md を改善
