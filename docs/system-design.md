## 投稿時刻のランダム化方針

人間が起きている時間帯に、不均等な間隔で投稿する。

### 重み付き時間帯

| 時間帯 | 重み |
|---|---|
| 7-9時（通勤） | 2 |
| 12-13時（昼休み） | 3 |
| 15-17時（午後） | 1 |
| 19-23時（夜） | 4 |
| 23-25時（深夜） | 1 |

### ランダム化ルール

- 当日の投稿数は 1〜2件（週末は1件）
- 時間帯はランダム選択（上記重みに従う）
- 分・秒は完全ランダム
- 投稿間隔は最低15分以上空ける
- 月1〜2日は「投稿しない日」を入れる（人間っぽさ）

## コンテンツ生成のプロンプト設計

```python
prompt = f"""
あなたは私の代わりにThreadsに投稿します。

# 私のナレッジ（参照元）
{read_knowledge_file(selected_topic)}

# 私の口調・文体
{read_file("knowledge/voice.md")}

{read_json("knowledge/used_log.json")[-30:]}

# 今回の投稿条件
- フォーマット: {random.choice(["気づき系", "ノウハウ箇条書き", "問いかけ", "失敗談", "短文一言"])}
- 文字数: {random.choice([80, 150, 250, 400])}字程度
- 絵文字: {random.choice(["なし", "1個だけ", "2〜3個"])}
- 末尾CTA: {random.choice([True, False, False, False])}  # 25%だけCTA

# 出力ルール
- ハッシュタグ過剰NG（多くて1〜2個）
- 「いかがでしたか」「ぜひ」など量産系ワード禁止
- 1人の人間が思いついた瞬間の文体で
- ナレッジから一語一句抜き出さず、今日の自分の気分で書き直す
"""
```

## AI臭を消すチェックリスト

| AI臭の出やすい要素 | 対策 |
|---|---|
| 毎回同じ構成（結論→理由→まとめ） | 5〜7パターンをランダム選択 |
| 句読点が完璧 | 時々体言止め・倒置を許可 |
| 絵文字位置が一定 | 使う/使わない/位置をランダム |
| 全部「です・ます」or「だ・である」 | 投稿ごとにランダム切替 |
| 長さが均一 | 80字〜500字まで分布 |
| 改行スタイル一定 | 改行なし／2行ごと／3行ごとを混ぜる |

## ダッシュボード構成（Streamlit想定）

### 表示要素

1. KGI進捗カード（フォロワー、月リーチ、DM件数、商談化）
2. 日別リーチ推移（折れ線グラフ）
3. 軸別パフォーマンス（横棒グラフ）
4. 時間帯別エンゲージメント率（ヒートマップ）
5. 今週のトップ投稿 TOP5
6. Claudeによる週次インサイト（自動生成）

### Claude週次レポート自動生成

毎週日曜23時に以下を実行：

```python
weekly_data = get_last_7days_metrics()
prompt = f"""
直近1週間のThreadsデータです：
{weekly_data}

これまでの傾向と比較して、以下を出してください：
1. 良かった点（数値根拠つき）
2. 悪化した点
3. 来週の投稿戦略の修正提案（軸別の投稿配分）
4. 注目すべき投稿とその理由
"""
report = claude_api.send(prompt)
save_to_db(report)
```

## 重要な制約と禁止事項

### 公式APIの制限

- 投稿：24時間で250件まで
- 返信：24時間で1,000件まで
- 削除：1日100件まで

### 規約上の禁止事項（実装NG）

- 他人の投稿への自動いいね
- 自動フォロー・自動アンフォロー
- 他人の投稿への自動リプライ
- スクレイピング・非公式API利用

→ 自動化スコープは「自分の発信パイプライン」だけに限定する。
